Notion View 資料呈現方式

Notion Views​

Notion Database 功能的延伸，
允許 Notion Database 依不同的目的改變視覺呈現。
Database 預設以 Table View 呈現。

Views (Layout)​

新建立 Database 時會在主頁下自動建立 Database Page。
Page 預設會以 View of database 命名。建議即刻依 view + db 改名。
若新增其他 Database View，則 Database Page 下會出現相關子分支。

Database Page Menu 相關結構

基本設定:​

Database Layout 與 Columns 的顯示隱藏，相關設定在 右上的   …  
與 View 相關的 Specific 設定會放在 View 右上的相關快捷或直接點 View Title
Card 的相關設定或 peak 會在 card 右上方

Database Views 相關設定入口

  • Layout 設定

  • 單一 View 專屬設定

Peak 頁面:​

Peak 指的是 Row 的 Detail Page
當滑鼠移至 Name Column 時所開啟的 Detail Page。
這個 Page 是 Row 的一部分，不是 SubPage。
此外，資料庫頁面可以自行設定 Detail 頁呈現資訊與排版方式，
可能因部分的 View 在呈顯時僅顯示指定摘要資訊，故提供 Row Detail Page 功能。

Peak (Detail Page)

Detail Page UI 設定入口:​

  • 🖱hover > 移至標題上方 > Customize Layout
  • 或是開啟 peak page > 右上 … > Customize Layout
  • 拖拉 Panel 即可進行版面編排
  • 完成後記得右上角的 Apply to all pages 進行儲存

Peak (Detail Page) 畫面調整
